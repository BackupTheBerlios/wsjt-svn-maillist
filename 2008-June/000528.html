Log:
- spec162.f had an #ifdef removing the form="binary" if it wasn't being
  compiled with CVF defined. g95 with -cpp compiled without form="binary"
  fortran was not since spec162.f was lower cased .f extension. 
  binary is only really needed for Windows machines that have to translate
  cr/lf combinations, it is not needed on unix. This probably caused
  the problems with decode on unix with gfortran42.
- removed superfluous make commands as they were already all covered
  by the default compile rules
- I had missed a superfluous -ftrace=full in configure that is only
  for g95. removed.
